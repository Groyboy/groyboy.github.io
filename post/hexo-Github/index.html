使用Hexo+GitHub搭建个人博客


前提条件

首先在搭建博客之前你需要放松下来不需要紧张，即使没有任何的编程基础都是没有问题的，你只需要有一台mac电脑一切都不是问题。


为什么写这篇文章


在我搭建博客之前也是做过一点调查，大部分的人都喜欢使用hexo来搭建博客，因为搭建速度很快也支持markdown输入（你不用管markdown是什么你只要知道很简单就OK），网络上大部分的教程也是写的天花乱坠让人深陷迷谭，其实也不是他们的错，很多人写的文章早已经过时了有的甚至都可以刻上历史，在当下这个网络飞速发展的时代，不刷新恐怕只有等待淘汰的那一天，所以写这篇文章也是想让大家都绕开我之前遇到的坑


准备喝茶


在开始的时候我劝你在旁边准备一杯茶叶，和一本书，因为全程无尿点，怕你无聊咯，现在开始，现在你需要做的就是打开你的终端
没错就是他，别看它长得像块黑板其实大有用处，嘿嘿。
现在开始检测你的环境,你要在你的小黑板上先输入几个命令（如果你是纯小白那么久直接到安装教程去进行操作）

输入：node -v
v12.1.0(会出现这样的标示它是版本号而已区别不是很大）
输入：npm -v
6.9.0(与上面相似）
输入：Git –version
git version 2.21.0（同上）


安装环境



首先打开终端输入：ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/
install)”         //这一步是来安装homebrew的如果你已经安装过了可以食用brew -v来查看
其次输入：brew install node        //这一步是来安装node和npm的安装之后你可以用上面的步骤来查看是否成功
安装Git：brew install git        //同上安装之后可以查看是否成功<

安装过程会有一点慢如果这时电脑发热是正常现象你不用惊慌


开始搭建


在搭建开始你需要安装最后一样东西 那就今天的主角“hexo”

sudo npm install -g hexo    //当你安装完成之后你一样可以使用hexo -v来查看是否完成

现在需要在终端输入
mkdir myblog  //它会在你的根目录下创建一个叫做myblog的文件夹你可以在访达中找到它

现在输入
cd myblog 这条命令会让你进入这个文件夹

当你进入之歌文件夹之后会看到这样的标示。

现在开始初始化
使用：hexo init
在文件夹内安装npm：sudo npm install

做完这些之后你就可以使用
输入：hexo d       //把部署好的内容生成到博客
输入：npm install hexo-server –save     //安装一下hexo他们家的独立服务器
输入：hexo s       //这时候会出现一个http链接你只需要把链接输入到浏览器就可以看到刚刚建立的blog


这个就是刚刚建立好的blog。


连接GitHub服务器


做完上面这些你算是已经开了一个头了，现在开始注册GitHub账号，你只需要有一个邮箱即可，点击GitHub传送门。
登陆之后在点击你的主页右上角+号选择New repository，然后在Repository name下面填写你”GitHub账号”.github.io
其他的如果看不懂就直接选择create repository这个时候你就成功创建完成了你的GitHub仓库
现在回到你的终端还是你的myblog文件夹内

输入：git init   //初始化git仓库
然后输入：vim _config.yml   //如果你有编码软件就可以使用编码软件打开文件夹在这里我推荐Sublime编码软件

当你进入vim编码时没用过也不用惊慌,把输入法转换成英文输入法然后按下i键你的左下角会提示
–INSERT–这时候翻到文本底部找到deploy
  deploy:
  type: git
  repository: https://github.com/Groyboy/groyboy.github.io.git   //将它改为你的仓库url
  branch: master       //这里改成master主分支

你的url在这里找到只需要复制
过来就可以了
现在执行：hexo d
         hexo g

若执行hexo g出错则执行npm install hexo –save，若执行hexo d出错则执行npm installhexo-deployer-git–save。错误修
正后再次执行hexo g和hexo d上传到服务器。
如果你是第一次关联GitHub的话会提示你输入你的账号 和密码你只需要根据提示进行输入即可，这个时候就可以通过
https://groyboy.github.io/来访问你的blog了。


使用ssh到GitHub


如果你觉得每次提交都要输入账号密码太麻烦的话这个时候你就需要生成你电脑的ssh密码了

输入：cd – //退回到你的根目录
再次输入：ssh-keygen -t rsa -C “your_email@exampl”       //这里面改成你注册GitHub时所用的邮箱
然后输入：open ~/.ssh  //之后会弹出访达ssh所在的路径

生成后会有一个.ssh文件里面会生成id_rsa.pub和id_rsa,“id_rsa”是私钥(不再特殊情况下不要透露私钥），“id_rsa.pub”是公共钥
匙打开id_rsa.pub打开GitHub官网，点击你的头像你会看到settings,选择SSH and GPG keys点击New ssh key，在title中输入你
的电脑型号来提示你时那台电脑的ssh，在key中粘贴刚刚复制的id_rsa.pub公匙然后点击add ssh key然后就添加完成啦


配置hexo主题


点击hexo跳转到hexo官网在左上方选择主题（如果你是英文界面可以在右上方修改中文）
选择你喜欢的主题点击你喜欢主题标题（是标题不是图片！）会自动跳转到博主的GitHub上点击clone or download然后复制链接回到
你的myblog文件夹内

输入：git clone “你选择的主题GitHub链接” themes/“主题的名字”

举个例子：git clone https://github.com/Haojen/hexo-theme-Anisina.git themes/Anisina
这是haojen先生所做的Anisina博客 做完之后
输入：vim _config.yml

还是和上次一样找到“theme:”在后边添加一个空格并且写上你刚刚下载的主题博客名字，然后输入hexo clean && hexo d && hexo g
，回到你的blod页面点击刷新，如果这时候没有加载过来不用着急，打开你的浏览设置清理一下历史记录，然后再次点击刷新这个时候就会加载完成


最后总结


最后还是想提醒大家一下教程虽然不够完整，但是你按照我说的做定然也会八九不离十了，如果在操作过程中出现错误可以及时翻阅文档
上网查询查看别人做错之后如何修改，同时也希望你能在写博客的路上坚持下去，到这里才是真正的开始，如果你觉得GitHub服务器有点慢
那你可以到阿里云哪里买一台服务，并且配置一个域名（当然你也可以不配置，如果你按时交网费的话加载问题不会很卡）如果有不是很明白的可以通过我的介绍里面找到的联系方式，我会为您排出不必要的麻烦，thinks you over
